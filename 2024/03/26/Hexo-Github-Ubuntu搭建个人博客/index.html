环境准备

Node.js

Node.js 是一个基于 Chrome V8 引擎的、由C++开发的 Javascript 运行环境。

Node.js的诞生可以追溯到2009年，当时由Ryan Dahl在JSConf会议上首次展示了Node.js。Node.js最初是作为一个实验性的项目开始的，旨在提供一个基于事件驱动、非阻塞I/O的JavaScript运行环境，以便开发高性能的网络应用程序。

Ryan Dahl当时对传统的服务器模型（如Apache）存在的性能瓶颈和开发效率问题[^1]感到不满，希望能够提供一种更高效的解决方案。他基于V8引擎（由Google Chrome浏览器使用的JavaScript引擎）开发了Node.js，使得JavaScript不仅可以在浏览器中运行，还可以在服务器端运行，从而实现了全栈JavaScript的愿景。

Node.js的出现极大地推动了JavaScript在服务器端的应用，使得开发人员可以使用同一种语言进行前后端开发，简化了开发流程并提高了开发效率。随着时间的推移，Node.js逐渐成为了一个流行的服务器端运行环境，并且在Web开发领域发挥着越来越重要的作用。

Node.js 最初的定位是提升 Ryan 自己的日常工作效率，也就是用来写服务器代码的，然而后来 Node.js 在前端领域却大放异彩。虽然，Web 前端的JS代码最终还是运行在浏览器中。但是，Node.js 诞生以后，前端大爆发，类似 React/Vuejs 这样的前端框架的开发环境变得非常强大，Node.js 是这些开发环境运行的基础。

node.js安装

sudo apt install nodejs

  • 每次请求都会创建一个新的线程或进程：在传统的服务器模型中，每当有新的请求到达
    时，服务器会为其创建一个新的线程或进程来处理。这种模式在高并发情况下会导致大量
    的线程或进程被创建，消耗大量的系统资源，降低了服务器的性能。

  • 阻塞I/O操作：传统服务器模型中的I/O操作通常是阻塞的，即当服务器执行一个I/O操作
    时，它会一直等待直到该操作完成。这种阻塞模式会导致服务器在等待I/O操作完成时无法
    处理其他请求，降低了服务器的并发处理能力。

  • 多线程编程难度大：在传统的服务器模型中，多线程编程是必不可少的，而多线程编程
    往往会引入诸多问题，如死锁、竞态条件等，增加了开发的复杂度和难度。

  • 语言不一致：在传统的服务器模型中，通常会使用不同的语言来开发前端和后端，如
    HTML/CSS/JavaScript用于前端，而PHP/Java/Python等用于后端，这种语言不一致会增加
    开发人员的学习成本和沟通成本。

npm

npm是Node.js的包管理工具，全称Node Package Manager。它是Node.js生态系统中的一个
重要组成部分，用于帮助开发人员安装、管理和共享JavaScript包和依赖项。

安装

sudo apt install npm

安装碰到的问题以及解决

  • 提示bash: /usr/bin/node: cannot execute binary file: Exec format error
    一般是安装的版本错了或者网络有问题导致安装失败。重新安装即可。
  • EACCESS权限错误
    可以参考npm官方文档的解决方法
